# 达人营销执行看板 (Influencer Marketing Execution Dashboard) 

## 1. 项目立项说明 (Project Charter)

### 1.1 项目背景
在达人营销（媒介采购、市场推广）业务中，团队常常面临资源触达进度不透明、业务流转节点卡顿、团队负载分配不均以及招募任务进度难以全局追踪等痛点。为了提升团队的执行效率和协同能力，需要一个统一的执行看板，让各级角色（媒介、市场、团队Leader）都能清晰地掌握个人和团队的工作状态。

### 1.2 项目目标
打造一个多视角的“执行看板”，提供：
- **资源池与数据监控**：实时查看线索、达人、邮件触达及合作转化的核心数据。
- **业务流转链路**：监控从提报、报价、合作确认、执行（审稿）、到最终结算打款的全链路待办情况，区分“我处理”与“待对方处理”。
- **团队负载均衡**：可视化团队各成员的待办积压分布，辅助Leader进行资源与任务调度。
- **任务目标追踪**：按项目维度的招募进度管理，直观呈现目标达成率和剩余时间。

### 1.3 目标用户
- **一线执行人员（媒介/市场）**：使用“我的工作台”视角，聚焦个人业绩、待办任务以及个人负责项目的进度。
- **团队管理者（Leader）**：使用“团队全局视图”，可以按部门/组织架构筛选成员，统筹团队整体的工作负载、审批瓶颈以及整体目标的达成进度。

## 2. 产品需求文档 (PRD)

### 2.1 全局交互与架构
- **视图切换**：支持“我的工作台”（self，个人视角）和“团队全局”（team，管理视角）的无缝切换，切换按钮固定居中于顶部导航，右侧组件显隐不影响其位置。
- **角色切换**：支持“媒介”与“市场”角色的切换，不同角色对应的核心节点（媒介重执行跟进，市场重提报审批）数据模型不同；切换角色时清空成员筛选。
- **组织架构人员筛选（组件：MemberSelect）**：
  - 仅在“团队全局视图”下显示，位于主内容区顶部的“成员筛选”条内。
  - 按当前角色只展示对应部门（媒介→媒介部，市场→市场部）及成员，支持部门全选/半选（indeterminate）。
  - 筛选结果实时联动统计卡片、业务链路、团队负载分布图。
  - 数据权限提示“仅可查看自己及下级成员数据”独立显示在筛选条右侧，不放在下拉项内。
- **字段说明入口**：顶部导航提供“字段说明”按钮（BookOpen 图标），点击在新标签页打开 `DATA_SPEC.md`（数据接入说明文档），便于查看者理解各指标含义与口径。

### 2.2 核心功能模块

#### 模块一：资源池与触达数据 (StatCard，按岗位差异化)
- **媒介角色**：
  - 资源池：线索总量、本月新增线索、达人总量、本月新增达人。
  - 合作转化：累计合作次数、本月新增合作、累计执行次数、本月新增执行、待执行。
  - 邮件触达：本月发送邮件、本月接收邮件、本月触达邮箱。
- **市场角色**：
  - 资源池：招募任务总数、本月新增招募任务。
  - 合作转化：累计合作次数、本月新增合作、累计执行次数、本月新增执行、待执行。
- **数据联动**：self 视图展示当前用户数据；team 视图展示全队或选中成员的聚合数据（待执行从 WorkflowStats 派生）。
- 每组指标一行四个（响应式 grid-cols-4）。

#### 模块二：图表概览区
- **趋势分析图 (TrendChart)**：展示核心指标的历史走势，支持 7d/14d/30d 切换；媒介与市场视角均展示提报数、执行数、确认合作三条线，图例与悬浮顺序固定为 提报数->执行数->确认合作。
- **个人备忘录 (PersonalMemo)**：self 视图下，展示当前登录用户的备忘便签（localStorage 持久化）。
- **团队负载分布图 (TeamWorkloadChart)**：team 视图下显示。
  - 水平堆叠柱状图，X 轴为成员，各项待办任务积压数量堆叠。
  - 横向滚动 + 动态宽度（每成员至少 60px），保证柱子可读。
  - 角色维度隔离（媒介/市场节点不同），受成员筛选联动。
  - 仅悬浮 Tooltip 展示明细，无底部图例。
  - 点击成员柱可在新标签页打开该成员的个人详情视图（团队视角按该成员筛选，不改当前页主筛选）。

#### 模块三：核心业务流转链路 (WorkflowPipeline + StageDetailModal)
- **展示形式**：三阶段链路看板——投放前 / 投放中 / 投放后，整体包在“业务流转链路”模块容器内。
- **节点标注**：每个节点标注负责角色，当前角色=「我处理」（琥珀色高亮），另一角色=「待对方」（灰色），已完成=绿色。
- **点击查看详情**：点击任一节点弹出明细弹窗（StageDetailModal），展示该节点积压单据列表（项目/负责人/待处理数/去处理或去查看），便于跟进与执行。明细弹窗为看板形式的概览，不在此汇总各执行项的停留天数（停留天数在实际操作页查看）；亦不在弹窗内一键催办，需进入实际详情按情况处理。
- **联动**：self 视图用个人 WorkflowStats，team 视图用聚合数据。

#### 模块四：招募任务进度追踪 (TaskProgressTable)
- **字段**：项目名称（含关注星标）、截止日期及剩余天数（紧急着色）、执行进度（完成/目标+缺口）、漏斗数据（提报/报价/确认合作/执行）、待办、操作。
- **待办列**：self 视图为「我的待办」，team 视图为「本组待办」，均显示待办数。
- **操作**：点击「待办」按钮展开该任务下的待办明细（待审稿/待审核支出），self/team 均支持展开。

## 3. 技术栈
- Vite + React 19 + TypeScript
- Tailwind CSS（亮色 slate 主题）
- Recharts（趋势线图 + 负载堆叠柱状图）
- Lucide React（模块标题图标，克制使用）
